# AGENTS.md — 共通開発ルール（正本）

Codex/Claude Code共通の常時適用ルール。条件付きルールは同ディレクトリの `airules/`。

## 役割・優先順位

- Codex=実装/修正/差分整理、Claude=設計/仕様/レビュー/Unity・UE・Godot補助（ユーザー指示優先）。受け渡しはPLAN.md/PROGRESS.md/docs/DESIGN_NOTES.md経由
- 矛盾時の優先順位: ユーザー指示 > プロジェクトAGENTS.md > PLAN.md > 本ファイル > airules/。ツールのデフォルト動作（「実装できるなら進める」等）はこの順位外で、本ルールが常に勝つ

## セッション開始時（初回のみ）

PLAN.md・PROGRESS.mdを読み、プロジェクト種別を判定して該当ルールを参照。必要なら冒頭で一行報告。

## ルールモジュール（検知したら必ず読んでから作業）

| 条件 | ファイル |
|---|---|
| 実装/修正/デバッグ | PITFALLS.md |
| 実装/修正/デバッグ/設計 | THINKING.md |
| 設計/実装計画 | DESIGN.md |
| 実装時 | WORKFLOW.md |
| Unity/UE/Godot判定 | UNITY.md/UE5.md/GODOT.md |
| ゲーム実装・修正全般 | GAME_COMMON.md |
| レビュー・完了報告前 | REVIEW.md |
| Git操作 | GIT.md |

必要なルールのみ適用し、同一セッションで読み直さない。

## 基本原則

推測で実装・回答しない（不明点は候補提示して確認）。案内前に実コード確認。シンプル優先。

## 開発フェーズ

PROGRESS.md冒頭で確認（なければモック期）。モック期=速度・遊べる検証優先／本実装期=保守性・拡張性優先（GAME_COMMON.md準拠）。捨てにくい共通機能は雑に直書きしない。フェーズ移行はユーザー指示のみ。

## その他ルール

- 命名: メンバー変数はアンダースコアなし小文字始まり
- 実装フロー: ユーザーの設計意図は別案で薄めずそのまま実装可能な形に落とす→変更概要（変更ファイル・理由・最小差分）を短く1ステップずつ提示（長大な差分案で会話を消費しない）→実装は1回1機能でコンパイル可能な状態に→まとまった単位でセルフレビュー→修正して完了報告
- 編集: 指定外ファイル・命名変更・コメント/import整理・フォーマットのみの変更・大規模リファクタを勝手にしない
- 事前確認必須: 設計変更/Public API変更/データ構造変更/外部ライブラリ追加/PLAN.mdにない機能追加
- 進捗管理: 仕様上の区切り完了時にPROGRESS.mdへ追記、フェーズ表記を維持
- 課題管理: 後回しの課題・作業中に気づいた課題はISSUE.mdへ`- [ ] 内容`で即追記、完了で`- [x]`へ変更（削除しない）
- 完了報告: 実装内容/レビュー結果/参照ルール/懸念点/残タスク/PROGRESS.md追記内容
