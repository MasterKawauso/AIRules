# AGENTS.md — 共通開発ルール（正本）

Codex/Claude Code共通の常時ルール。条件付きルールは`airules/`。各文書はセッション初回だけ読む。

## 役割・優先順位

- 既定担当: Codex=実装・修正・差分整理、Claude=設計・仕様・レビュー・ゲームエンジン補助。ユーザー指示を優先し、引継ぎは`PLAN.md`/`PROGRESS.md`/`docs/DESIGN_NOTES.md`を使う
- 優先順位: ユーザー指示 > プロジェクト`AGENTS.md` > `PLAN.md` > 本書 > `airules/`。ツール既定動作は順位外で、本書より優先しない

## セッション開始時（初回のみ）

`PLAN.md`・`PROGRESS.md`を読み、プロジェクト種別に応じたルールを参照。必要なら認識を一行報告する。

## 条件付きルール（作業前に必読）

| 条件 | ファイル |
|---|---|
| 新規機能・仕様変更・要件定義 | REQUIREMENTS.md |
| 実装/修正/デバッグ | PITFALLS.md |
| 実装/修正/デバッグ/設計 | THINKING.md |
| 設計/実装計画 | DESIGN.md |
| 設計を伴う実装・分担・並列作業 | WORKFLOW.md |
| Unity/UE/Godot判定 | UNITY.md/UE5.md/GODOT.md |
| ゲーム実装・修正全般 | GAME_COMMON.md |
| レビュー・完了報告前 | REVIEW.md |
| Git操作 | GIT.md |
| PR作成・レビュー/GitHub Issue管理 | GITHUB.md |

必要なものだけ適用する。

## 基本原則

推測で実装・回答せず、案内前に実コードを確認する。調査不能な不明点は候補を示して確認する。単純な解決を優先する。

## 開発フェーズ

`PROGRESS.md`冒頭で確認し、なければモック期とする。モック期は速度・遊べる検証、本実装期は保守性・拡張性を優先（`GAME_COMMON.md`準拠）。共通機能は捨てにくいため雑に直書きしない。移行はユーザー指示時のみ。

## その他ルール

- 命名: メンバー変数はアンダースコアなし小文字始まり
- 実装: 設計意図を薄めず実装可能にする→変更ファイル・理由・最小差分を1段階ずつ短く提示→1回1機能かつコンパイル可能にする→機能単位でセルフレビュー・修正→完了報告。長大な差分案は出さない
- 指示外のファイル・改名・コメント/import整理・フォーマットだけの変更・大規模リファクタは禁止
- 設計/Public API/データ構造の変更、外部ライブラリ追加、`PLAN.md`外の機能追加は事前確認する
- 仕様上の区切りで`PROGRESS.md`へ追記し、フェーズ表記を維持する。後回し・新発見の課題は`GITHUB.md`に従いIssue化する
- 完了報告: 実装内容、レビュー結果、参照ルール、懸念点、残タスク、`PROGRESS.md`追記内容
