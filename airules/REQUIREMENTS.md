# 要件発掘・要件確認ルール

新規機能、仕様変更、複数画面・複数状態にまたがる実装の前に適用。
軽微な不具合修正、文言変更、明確な単一変更は対象外。

## 目的

ユーザーが明示した要求だけでなく、
機能成立に必要だが未定義の要件を設計前に検出。

既存の要求から勝手に仕様を確定せず、
不足・選択肢・対象外を区別。

## 手順

1. 明示された要求と目的を整理する
2. アクター、操作、状態、データを列挙する
3. 正常系以外の経路を確認する
4. 未定義事項を分類する
5. 必須事項を解消してから設計へ進む

## 必須観点

- 初回利用、通常利用、再利用
- 成功、失敗、キャンセル、再試行
- 連打、二重実行、同時操作
- 開始前、処理中、完了後
- 画面遷移中、ロード中、終了中
- 権限、役職、所有者ごとの差
- データの生成、更新、保存、破棄、復元
- エラー表示とユーザーへの通知
- 中断後に復帰できるか
- テスト、デバッグ、ログで確認可能か
- 実装対象と対象外の境界
- 既存機能との競合

## マルチプレイ時の追加観点

- ホストとクライアントの権限差
- 参加、退出、切断、再接続
- ホスト移譲
- 同時要求と処理順序
- RPCや通知の重複、遅延、順序逆転
- Sceneロード中の参加者変化
- 正規状態の所有者
- UIのローカル状態と同期状態の不一致
- タイムアウト後に成功通知が届く場合
- 一部クライアントだけ失敗した場合

## 分類

各項目を次のいずれかに分類。

1. 明示要件
2. 要件から必然的に導ける要件
3. 機能成立に必須だが未定義
4. ユーザー判断が必要
5. 今回は対象外にできる
6. 将来拡張にすぎない

3と4が未解消のまま設計を確定しない。
ただし実装を止めないよう、ユーザー判断が不要な安全な既定値がある場合は、
その根拠と変更可能箇所を明示して仮決定。

## 出力

- 目的
- 明示要件
- 潜在要件
- 未決事項
- 対象外
- 完了条件
