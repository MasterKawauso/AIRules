# WORKFLOW — 分担作業フロー

設計を伴う実装・タスク分割・並列実装時に適用。軽微な単発修正は対象外。各実装は`AGENTS.md`のフローに従う。

## 流れ

1. 開始時に「設計・分割・レビュー」「実装」「統合レビュー」各担当のAI・モデル・思考深度を選択式で確認する
2. 要件整理→要件レビュー→未決事項の裁定→設計→設計レビュー→分割→Worker実装（必要なら1〜5並列）→成果物別レビューと修正→統合→統合レビュー→完了(なおPLAN.mdが整理済みならば要件フローは省略、DESIGN.mdが整理済みなら設計フローを省略、不明ならユーザーに聞く)
3. 1 Workerにつき1 Reviewerを起動し、問題がなくなるまで修正する。ただしCodexは成果物別レビューを省き統合レビューだけ行う
4. 統合レビューは統合差分だけを対象にCodex↔Claudeで相互レビュー・修正し、問題を解消する
5. Claude→Codexは`codex:rescue`、Codex→Claudeは事前に`Get-Command claude`で確認後`claude -p "<レビュー依頼>"`。利用不能なら停止し、ユーザーへ相互レビューを依頼する

## 役割

- Architect: 設計・仕様整理・タスク分割・完了条件定義
- Worker: 通常より思考負荷を1段下げ、担当範囲だけ実装。範囲外の設計・仕様変更、大規模リファクタは禁止。不明点は報告
- Reviewer: 担当Workerの成果物だけを問題解消までレビューし、必要なら改善案を示す
- Integrator: 統合、接続漏れ・競合・仕様差異の確認、必要最小限の統合作業、最終レビュー

レビューは`REVIEW.md`、設計は`DESIGN.md`を正本とする。
