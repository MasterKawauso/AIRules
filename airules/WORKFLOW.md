# WORKFLOW — 分担作業フロー

設計を伴う実装・タスク分割・並列実装時に適用。軽微な単発修正には不要。各実装はAGENTS.mdの実装フロー（1回1機能・セルフレビュー）に従う。

## 流れ

設計 → 設計レビュー → タスク分割 → Worker実装（必要に応じ1〜5並列） → Worker成果物ごとにレビュー（問題がなくなるまでレビュー⇔修正） → 成果物統合 → 統合レビュー → 完了
(Codexの場合レビューは統合レビューのみとする)

## 役割

- Architect: 設計・仕様整理・タスク分割・完了条件定義
- Worker: 割り当て範囲のみ実装。思考負荷は通常より1段階下げる。担当外の設計変更・仕様変更・大規模リファクタ禁止。不明点は推測せず報告
- Reviewer: 担当Workerの成果物のみレビュー（他Workerは対象外）。問題がなくなるまでレビュー⇔修正を繰り返し、必要に応じ改善案を提示
- Integrator: 成果物統合・接続漏れ/競合/仕様差異の確認・必要に応じ軽微な統合作業のみ実施・最終レビュー

レビュー観点・形式はREVIEW.md、設計手順はDESIGN.mdが正本。
