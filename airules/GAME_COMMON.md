# GAME_COMMON — 共通モジュール設計ルール

ゲームプロジェクトでの実装・修正全般（エンジン問わず、新機能に限らない）に適用。エンジン非依存の方針。実装詳細はUNITY.md/UE5.md/GODOT.mdに従う。

## 基本方針

- ゲーム固有コードと共通機能を混ぜない（モック期は過剰な共通化をしない）
- 他ゲームでも使う可能性が高い機能のみ共通モジュール化する。2〜3回の重複だけで即共通化しない
- 実装速度を大きく落とす抽象化は避ける

## 共通モジュール対象（ゲーム固有コードへ直書きしない）

| 機能 | 内容 |
|---|---|
| シーン遷移 | ロード・アンロード・遷移制御 |
| フェード | 画面遷移時の演出 |
| サウンド管理 | BGM・SEの再生・停止・音量制御 |
| 設定管理 | 音量・解像度・言語等の保存・読込 |
| 多言語管理 | ローカライズ・言語切替（UIText追従含む） |
| セーブデータ管理 | 保存・読込・削除 |
| マスター管理 | アイテム・キャラ等の静的定義データ |
| Steam連携 | 実績・リーダーボード・ロビー・P2P・フレンド |
| マルチプレイ基盤 | セッション管理・同期・RPC基盤 |
| 入力管理 | キーボード・パッド・タッチ抽象化・リマップ |
| UI管理 | パネル開閉・スタック管理・ポップアップ |
| ローディング画面 | 非同期ロード中の表示・進捗管理 |
| エフェクト管理 | パーティクル・VFXのプール・再生・停止 |
| 効果音ボタン | タップ時にSEを鳴らす汎用ボタン |

## 設計ルール

- 依存方向: `ゲーム固有コード → 共通モジュール → エンジン/SDK`（逆転禁止）
- 共通モジュールは独立した名前空間（例`KawausoForge.Core`）に置きゲーム固有と混在させない
- Unityは`.asmdef`で依存関係を明示（循環禁止）
- 各モジュールはインターフェース（`ISceneManager`等）を公開し実装差し替え可能にする
- Steam連携はラッパー経由で呼ぶ。Unityは`CommonComponents`があれば優先使用

## 実装前チェックリスト

- [ ] ゲーム固有知識（キャラ名等）を共通モジュールに含めていないか
- [ ] 共通モジュールからゲーム固有クラスを参照していないか
- [ ] 別プロジェクトへそのままコピーして動くか
- [ ] 依存に循環がないか / インターフェース公開でモック差し替え可能か
