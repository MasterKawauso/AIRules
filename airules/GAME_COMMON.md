# GAME_COMMON — 共通モジュール設計ルール

全ゲーム実装・修正に適用するエンジン非依存方針。詳細は`UNITY.md`/`UE5.md`/`GODOT.md`。

## 基本方針

- ゲーム固有コードと共通機能を分ける。他ゲームでも使う可能性が高いものだけ共通化し、モック期や2〜3回の重複だけでは共通化しない
- 実装速度を大きく落とす抽象化は避ける

## 共通モジュール対象（ゲーム固有コードへ直書きしない）

| 機能 | 内容 |
|---|---|
| シーン遷移/フェード | ロード・アンロード・遷移・画面演出 |
| サウンド/効果音ボタン | BGM・SE・音量・汎用SEボタン |
| 設定/多言語 | 音量・解像度・言語の保存、ローカライズ、UIText追従 |
| セーブ/マスター | セーブCRUD、アイテム・キャラ等の静的定義 |
| Steam/マルチプレイ | 実績・ランキング・ロビー・P2P・フレンド・セッション・同期・RPC |
| 入力 | キーボード・パッド・タッチ抽象化・リマップ |
| UI/ローディング | パネル・スタック・ポップアップ・非同期進捗表示 |
| エフェクト | パーティクル・VFXのプール・再生・停止 |

## 設計ルール

- 依存方向: `ゲーム固有コード → 共通モジュール → エンジン/SDK`（逆転禁止）
- 共通モジュールは独立名前空間（例`KawausoForge.Core`）に置く。Unityは`.asmdef`で循環のない依存を明示する
- 本実装期までにinterface（例`ISceneManager`）を公開し差替可能にする。モック期は任意
- Steam連携はラッパー経由で呼ぶ。Unityは`CommonComponents`があれば優先使用

## 実装前チェックリスト

- [ ] ゲーム固有知識（キャラ名等）を共通モジュールに含めていないか
- [ ] 共通モジュールからゲーム固有クラスを参照していないか
- [ ] 別プロジェクトへそのまま移せるか
- [ ] 循環依存がなく、interfaceでモック差替可能か（本実装期のみ）
