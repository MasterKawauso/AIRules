# GITHUB — GitHub運用ルール

PR作成・レビュー対応・Issue管理時に適用。

## プルリクエスト

### 記載内容

- 日本語タイトル: `[関連機能] 作業内容`
- 本文: 概要/バグ原因（該当時）/クラス・ファイル別変更/テスト/残課題

### PR作成後のレビュー対応

1. ユーザーの「コメント完了」後に確認し、必須修正/任意改善/質問/対応不要へ分類する
2. 方針整理→修正→追加commit・push→返信を、問題解消まで繰り返す
3. マージはユーザーが行う。指示なくマージしない

## Issue

- 後回しの課題・作業中に気づいた課題はGitHub Issueで管理
- タイトル・詳細・ラベル（Plan/Feature/Bug/Refactor/Test/Else）を記載し、ラベルがなければ作る
- 作成不能ならタイトル・詳細・ラベル案を示し、ユーザーへ依頼する
