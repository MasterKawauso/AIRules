# GITHUB — GitHub運用

PR作成・レビュー対応・Issue管理時に適用。

## PR

- 日本語タイトル: `[関連機能] 作業内容`
- 本文: 概要、バグ原因（該当時）、主なファイル別変更、テスト、残課題。空の項目や差分の逐語説明は省く
- ユーザーの「コメント完了」後にレビューを確認し、必須修正/任意改善/質問/対応不要へ分類する。必要分だけ方針→修正→commit/pushし、必須でGitHubで返信する、未解決コメントを再確認する
- レビュー対応作業は別コミットに分ける
- マージはユーザーが行う。指示なく実行しない

## Issue

今回扱わない実行可能な課題だけ、既存Issueとの重複を確認して登録する。タイトル、再現・背景、完了条件、ラベル（Plan/Feature/Bug/Refactor/Test/Else）を記し、ラベルがなければ作る。作成不能なら同内容の案をユーザーへ渡す。
