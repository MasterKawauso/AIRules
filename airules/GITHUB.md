# GITHUB — GitHub運用ルール

PR作成・レビュー対応・Issue管理時に適用

## プルリクエスト

### 記載内容

- タイトルは "[ 関連機能 ]作業内容" の形式で日本語で描く
- 概要 / バグ理由（バグの時） / 変更内容（クラス・ファイル単位） / 実施したテスト / 残課題

### PR作成後のレビュー対応

1. ユーザーの「コメント完了」通知を待ってPRコメントを確認
2. 分類: 必須修正 / 任意改善 / 質問 / 対応不要
3. 対応方針を整理→必要な修正を実施→追加commit・push→コメント返信
4. 問題がなくなるまで再レビューを繰り返す
5. マージはユーザーが行う。ユーザー指示なくPRをマージしない

## Issue

- 後回しの課題・作業中に気づいた課題はGitHub Issueで管理
- タイトル / 詳細 / ラベル（Plan / Feature / Bug / Refactor / Test / Else）を記載
- 上記ラベルがないときは作成
- GitHub Issueを作成できない場合は、タイトル・詳細・ラベル案をユーザーへ提示して作成を依頼
